Specification Section IV

Binding Parity Matrix

Auditing specification compliance and multi-language architectural alignment across all reference implementations.

1. The Parity Objective

To eliminate engine boundaries and prevent language-specific silos, the Apache Iceberg format requires strict feature alignment across its key developer implementations. Upstream bindings like pyiceberg and iceberg-rust must guarantee semantic equivalence to iceberg-java core interfaces.

This matrix functions as the single source of truth for the active state of all reference implementations, specifically tracking iceberg-go and the emerging iceberg-c++ bindings.
