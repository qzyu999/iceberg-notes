Schema & Spec Evolution

Understanding metadata-only inplace schema modifications and partition specification transformations.

II. Schema & Partition Spec Evolution Mechanics

In traditional table formats, renaming, adding, or deleting fields requires a full rewrite of physical data files. Apache Iceberg guarantees metadata-only, inplace schema and partition spec updates.

By relying on Axiom 3 (Field ID Stability), the schema decouples logical column names from the underlying physical structures. Click the tabs in the code sandbox below to see the precise JSON state transition when renaming user_id to account_id and adding a new column last_login:

JSON Schema Metadata Transition
{
  "schema-id": 0,
  "type": "struct",
  "fields": [
    { "id": 1, "name": "user_id", "required": true, "type": "long" },
    { "id": 2, "name": "email", "required": false, "type": "string" }
  ]
}
{
  "schema-id": 1,
  "type": "struct",
  "fields": [
    /* Column 1 is renamed but preserves Field ID 1. Data files are NOT touched. */
    { "id": 1, "name": "account_id", "required": true, "type": "long" },
    { "id": 2, "name": "email", "required": false, "type": "string" },
    /* Column 3 is added with a newly allocated Field ID 3. */
    { "id": 3, "name": "last_login", "required": false, "type": "timestamptz" }
  ]
}

Similarly, when partition specs evolve (e.g. daily partitions become too large and require hourly partitions), existing data files remain unchanged. The table metadata tracks multiple active partition specifications, mapping each data file to its respective spec-id:

Partition Spec Configuration
{
  "default-spec-id": 0,
  "partition-specs": [
    {
      "spec-id": 0,
      "fields": [
        {
          "source-id": 4,          /* timestamp column */
          "field-id": 1000,
          "name": "event_date",
          "transform": "day"       /* Partitioned by date */
        }
      ]
    }
  ]
}
{
  "default-spec-id": 1,
  "partition-specs": [
    {
      "spec-id": 0,
      "fields": [
        { "source-id": 4, "field-id": 1000, "name": "event_date", "transform": "day" }
      ]
    },
    {
      "spec-id": 1,
      "fields": [
        /* Evolved Spec 1 splits events by hour of day for higher write volume */
        {
          "source-id": 4,
          "field-id": 1001,
          "name": "event_hour",
          "transform": "hour"
        }
      ]
    }
  ]
}
