Layered Architecture & Coordination

An exhaustive, under-the-hood analysis of the Catalog, Metadata, and Data layers, tracing the execution paths of distributed queries and transactional writes.

I. The Three-Tier Architectural Specification

Apache Iceberg is structured into three strictly decoupled layers: the Catalog Layer, the Metadata Layer, and the Data Layer. This modular design isolates catalog tracking from physical indexing, enabling lock-free transactional edits, schema/partition evolution without rewrites, and massively parallelized queries across multiple computing engines.

💡 Table Layers: The Real Estate Registry Analogy
Imagine buying and owning a parcel of land in a city:
• The Catalog Layer (The Land Registry Office): This is the central city hall registry containing a single book. The book maps a parcel number (table ID) to a specific ledger ID. City hall does not hold the actual land or blueprints—it only coordinates ownership swaps atomically.
• The Metadata Layer (The Blueprints & Surveys): This is the detailed surveyor ledger. It does not contain the dirt or grass itself. Instead, it holds precise mathematical maps, bounding boxes, and list addresses specifying exactly where the fence lines are located.
• The Data Layer (The Physical Land): The actual physical soil, grass, and trees residing at the physical GPS coordinates in the ground (the raw Parquet files in S3).
When a building inspector (query engine) scans the land, they first get the survey ID from city hall (Catalog), read the boundary maps (Metadata), and then walk directly to the physical soil blocks (Data).

II. The Detailed Coordination Paths

1. The Read Execution Path (Querying Data)

When an engine (like Spark, Trino, or PyIceberg) executes a query (e.g. SELECT * FROM db.events WHERE timestamp >= '2026-05-25'), it coordinates across the three layers:

  1. Catalog Resolution: The engine contacts the **Catalog** (e.g. REST Catalog) with the table identifier db.events. The catalog responds with the URI of the current root metadata file: s3://bucket/metadata/v4.metadata.json.
  2. Metadata Root Parsing: The engine reads the **Metadata JSON** from S3. It parses the schema, active partition specs, and finds the current Snapshot ID (e.g. snap_804) and its corresponding **Manifest List** Avro file path.
  3. Manifest List Filtering (Manifest Pruning): The engine reads the **Manifest List**. It evaluates each manifest file entry record. Since the manifest list annotates each manifest with its partition value bounds, the engine skips any manifest files whose partitions do not overlap with our query filter (Hour/Day Partition Pruning).
  4. Manifest File Parsing (File Pruning): The engine reads the remaining **Manifest Files** in parallel. It evaluates each manifest entry. Using the column minimum, maximum, and null counts stored in each manifest entry, the engine skips individual data files that cannot mathematically contain matching rows (Column Statistics Pruning).
  5. Physical Data Reading: The engine receives a highly refined list of physical data file paths (e.g., data-01.parquet, data-02.parquet). It bypasses all metadata layers and reads the raw row blocks directly from the **Data Layer** (cloud storage), splitting the files into parallel task threads.

2. The Write Execution Path (Atomic Commits & Retries)

When an ingestion writer appends new records, it coordinates across the layers to write new files and swap state pointers atomically:

  1. Data File Ingest: The writer engine streams input rows, buckets them according to the active partition spec, and writes brand-new, immutable Parquet files directly to the **Data Layer** (cloud storage).
  2. Manifest Serialization: The writer compiles the paths, row counts, and detailed column-level min/max statistics of the newly written files, and serializes a new **Manifest File** (Avro) containing these entries, marked with status = 1 (ADDED).
  3. Manifest List Synthesis: The writer creates a new **Manifest List** file. It copies all manifest pointers from the previous active snapshot, and appends a pointer to the newly created manifest file.
  4. Metadata Root Generation: The writer generates a new root **Metadata JSON** file (e.g., v5.metadata.json). This file registers the new snapshot ID, links it to our new Manifest List, appends it to the table history array, and updates the active snapshot pointer.
  5. Atomic Catalog Swap (OCC): The writer attempts to register the new root pointer in the **Catalog Layer** by requesting a conditional atomic update: "Swap db.events pointer from v4.metadata.json to v5.metadata.json if and only if the current pointer is still v4.metadata.json."
  6. Conflict Resolution & Rebase: If a concurrent transaction committed v5.metadata.json first, the catalog conditional swap fails. The writer reads the new v5.metadata.json to detect conflicts. If no overlaps exist (e.g. the concurrent transaction appended to partition B while we appended to partition A), the writer rebases: it rebuilds its manifest list and metadata JSON on top of v5.metadata.json and attempts to commit again as v6.metadata.json!

IV. Comprehensive Layer Coordination Simulator

The following complete, self-contained Python program simulates the entire three-tier architecture of an Iceberg table, tracing how the Catalog, Metadata, and Data layers cooperate to execute isolated scans and resolve write conflicts under optimistic concurrency control (OCC): 

import time
from typing import List, Dict, Any, Tuple

# =====================================================================
# DATA LAYER: Simulates physical storage files on S3
# =====================================================================
class DataFile:
    def __init__(self, path: str, partition_val: int, records: List[Dict[str, Any]], col_min: int, col_max: int):
        self.path = path
        self.partition_val = partition_val
        self.records = records
        self.record_count = len(records)
        self.col_min = col_min
        self.col_max = col_max

class S3StorageBucket:
    def __init__(self):
        self.objects: Dict[str, Any] = {}

    def write(self, path: str, payload: Any):
        self.objects[path] = payload
        print(f"  [S3 WRITE] {path}")

    def read(self, path: str) -> Any:
        if path not in self.objects:
            raise FileNotFoundError(f"S3 Object not found at {path}")
        return self.objects[path]

# =====================================================================
# METADATA LAYER: Indexes, Manifests, and Metadata JSONs
# =====================================================================
class ManifestEntry:
    def __init__(self, status: int, sequence_number: int, data_file: DataFile):
        self.status = status # 0: EXISTING, 1: ADDED, 2: DELETED
        self.sequence_number = sequence_number
        self.data_file = data_file

class ManifestFile:
    def __init__(self, path: str, entries: List[ManifestEntry], min_partition: int, max_partition: int):
        self.path = path
        self.entries = entries
        self.min_partition = min_partition
        self.max_partition = max_partition

class ManifestList:
    def __init__(self, manifest_files: List[ManifestFile]):
        self.manifest_files = manifest_files

class TableMetadataJSON:
    def __init__(self, format_version: int, current_snapshot_id: int, snapshot_history: List[Dict[str, Any]]):
        self.format_version = format_version
        self.current_snapshot_id = current_snapshot_id
        self.snapshots = snapshot_history # Maps snapshot_id -> manifest_list_path

# =====================================================================
# CATALOG LAYER: Maps table IDs to root pointers with atomic swaps
# =====================================================================
class RESTCatalog:
    def __init__(self, s3_bucket: S3StorageBucket):
        self.s3_bucket = s3_bucket
        self.table_registry: Dict[str, str] = {} # table_id -> current_metadata_path
        self.version_counters: Dict[str, int] = {} # table_id -> version count

    def register_table(self, table_id: str, metadata_path: str):
        self.table_registry[table_id] = metadata_path
        self.version_counters[table_id] = 1

    def get_table_root_pointer(self, table_id: str) -> str:
        if table_id not in self.table_registry:
            raise KeyError(f"Table {table_id} is not registered in catalog.")
        return self.table_registry[table_id]

    def commit_swap(self, table_id: str, expected_path: str, new_path: str) -> bool:
        """Simulates atomic conditional check-and-put swap (OCC)."""
        current = self.table_registry.get(table_id)
        if current != expected_path:
            # Concurrency conflict detected! Expected old version is stale
            print(f"  [CATALOG REJECT] Expected base '{expected_path}' but registry has evolved to '{current}'!")
            return False
            
        # Atomic swap succeeds
        self.table_registry[table_id] = new_path
        self.version_counters[table_id] += 1
        print(f"  [CATALOG SWAP SUCCESS] {table_id} pointer swapped atomically to: {new_path}")
        return True

# =====================================================================
# COORDINATOR: Simulates Reader/Writer Query Engines
# =====================================================================
class IcebergEngineCoordinator:
    def __init__(self, catalog: RESTCatalog, s3_bucket: S3StorageBucket, table_id: str):
        self.catalog = catalog
        self.s3_bucket = s3_bucket
        self.table_id = table_id

    def execute_query(self, filter_partition: int, filter_key_min: int) -> List[Dict[str, Any]]:
        """
        Simulates the entire READ coordination path:
        Catalog -> Metadata JSON -> Manifest List -> Manifest -> Data File
        """
        print(f"\n--- Starting Read Scan for '{self.table_id}' where partition={filter_partition} and ID >= {filter_key_min} ---")
        
        # 1. Resolve Root Pointer from Catalog Layer
        root_metadata_path = self.catalog.get_table_root_pointer(self.table_id)
        print(f"Step 1: Catalog resolved table ID to root path: {root_metadata_path}")
        
        # 2. Parse Metadata JSON
        meta: TableMetadataJSON = self.s3_bucket.read(root_metadata_path)
        print(f"Step 2: Read Metadata JSON. Current Snapshot ID: {meta.current_snapshot_id}")
        
        # Resolve active snapshot manifest list path
        manifest_list_path = next(
            snap["manifest_list"] for snap in meta.snapshots if snap["id"] == meta.current_snapshot_id
        )
        
        # 3. Parse and Prune Manifest List
        manifest_list: ManifestList = self.s3_bucket.read(manifest_list_path)
        print(f"Step 3: Read Manifest List containing {len(manifest_list.manifest_files)} manifest paths.")
        
        active_manifests: List[ManifestFile] = []
        for m_file in manifest_list.manifest_files:
            # Check partition pruning
            if m_file.min_partition <= filter_partition <= m_file.max_partition:
                print(f"  - Manifest '{m_file.path}' matches partition filter [{m_file.min_partition}, {m_file.max_partition}]. ✅")
                active_manifests.append(m_file)
            else:
                print(f"  - Manifest '{m_file.path}' bounds [{m_file.min_partition}, {m_file.max_partition}] do not match. Pruned! ❌")
        
        # 4. Parse Manifest Files and evaluate Column Stats
        print("Step 4: Evaluating manifest entries and column statistics bounds...")
        files_to_read: List[DataFile] = []
        
        for manifest in active_manifests:
            for entry in manifest.entries:
                if entry.status == 2: # Deleted
                    continue
                d_file = entry.data_file
                # Check column statistics bounds pruning
                if d_file.col_max >= filter_key_min:
                    print(f"  - File '{d_file.path}' has max key {d_file.col_max} >= filter {filter_key_min}. Matches! ✅")
                    files_to_read.append(d_file)
                else:
                    print(f"  - File '{d_file.path}' has max key {d_file.col_max} < filter {filter_key_min}. Pruned! ❌")
                    
        # 5. Read physical data files directly from Data Layer
        print(f"Step 5: Reading raw rows from {len(files_to_read)} matched physical data files directly.")
        final_rows = []
        for df in files_to_read:
            # Evaluate records
            for record in df.records:
                if record["id"] >= filter_key_min:
                    final_rows.append(record)
                    
        print(f"Scan complete. Retained {len(final_rows)} active rows.")
        return final_rows

    def execute_write_append(self, partition_val: int, raw_records: List[Dict[str, Any]], col_min: int, col_max: int):
        """
        Simulates the entire WRITE coordination path with OCC retry loops:
        Data Append -> Manifest -> Manifest List -> Metadata JSON -> Catalog atomic swap
        """
        print(f"\n--- Starting Transaction Append for '{self.table_id}' (Partition={partition_val}) ---")
        
        retry_count = 0
        max_retries = 3
        
        while retry_count < max_retries:
            # Read current base path
            base_metadata_path = self.catalog.get_table_root_pointer(self.table_id)
            meta: TableMetadataJSON = self.s3_bucket.read(base_metadata_path)
            
            # Step 1: Write raw data files directly to Data Layer
            file_id = int(time.time() * 1000)
            data_path = f"s3://bucket/data/partition_x={partition_val}/data_{file_id}.parquet"
            data_file = DataFile(data_path, partition_val, raw_records, col_min, col_max)
            self.s3_bucket.write(data_path, data_file)
            
            # Step 2: Write new manifest indexing our data file
            manifest_path = f"s3://bucket/metadata/manifest_{file_id}.avro"
            entry = ManifestEntry(status=1, sequence_number=meta.current_snapshot_id + 1, data_file=data_file)
            new_manifest = ManifestFile(manifest_path, [entry], partition_val, partition_val)
            self.s3_bucket.write(manifest_path, new_manifest)
            
            # Step 3: Synthesis new manifest list combining old manifest list and our new manifest
            base_snap_list_path = next(
                snap["manifest_list"] for snap in meta.snapshots if snap["id"] == meta.current_snapshot_id
            )
            base_snap_list: ManifestList = self.s3_bucket.read(base_snap_list_path)
            
            new_snap_list_path = f"s3://bucket/metadata/snap_{file_id}.manifest-list.avro"
            new_manifest_list = ManifestList(base_snap_list.manifest_files + [new_manifest])
            self.s3_bucket.write(new_snap_list_path, new_manifest_list)
            
            # Step 4: Write new root Metadata JSON
            new_snapshot_id = meta.current_snapshot_id + 1
            new_history = meta.snapshots + [{"id": new_snapshot_id, "manifest_list": new_snap_list_path}]
            
            new_meta_path = f"s3://bucket/metadata/v{new_snapshot_id}.metadata.json"
            new_metadata = TableMetadataJSON(format_version=2, current_snapshot_id=new_snapshot_id, snapshot_history=new_history)
            self.s3_bucket.write(new_meta_path, new_metadata)
            
            # Step 5: Attempt atomic pointer swap in Catalog Layer
            print("Step 5: Attempting atomic conditional commit swap in catalog...")
            success = self.catalog.commit_swap(self.table_id, base_metadata_path, new_meta_path)
            
            if success:
                print("Commit Transaction completed successfully!")
                return
            else:
                # Concurrent write collided! Increment retry count and loop
                retry_count += 1
                print(f"  [REBUILD & RETRY] Re-attempting ingest. Retry cycle: {retry_count}/{max_retries}")
                time.sleep(0.1)
                
        raise RuntimeError("Append transaction aborted after exceeding maximum concurrency retries.")

if __name__ == "__main__":
    # Initialize physical storage and catalog layers
    s3 = S3StorageBucket()
    catalog = RESTCatalog(s3)
    
    # 1. Setup Base Table (Snapshot ID 1) containing initial users
    # Data Layer: data_1.parquet (partition 1, keys 1-10)
    data1_records = [{"id": 1, "name": "Alice"}, {"id": 10, "name": "Bob"}]
    data_file_1 = DataFile("s3://bucket/data/partition_x=1/data_1.parquet", 1, data1_records, 1, 10)
    s3.write(data_file_1.path, data_file_1)
    
    # Metadata Layer: manifest_1.avro indexing data_file_1
    manifest_1 = ManifestFile("s3://bucket/metadata/manifest_1.avro", [ManifestEntry(0, 1, data_file_1)], 1, 1)
    s3.write(manifest_1.path, manifest_1)
    
    # Metadata Layer: snap_1 manifest list
    snap_list_1 = ManifestList([manifest_1])
    s3.write("s3://bucket/metadata/snap_1.manifest-list.avro", snap_list_1)
    
    # Metadata Layer: v1.metadata.json
    root_metadata = TableMetadataJSON(
        format_version=2, 
        current_snapshot_id=1, 
        snapshot_history=[{"id": 1, "manifest_list": "s3://bucket/metadata/snap_1.manifest-list.avro"}]
    )
    s3.write("s3://bucket/metadata/v1.metadata.json", root_metadata)
    
    # Catalog Layer: Register table pointer
    catalog.register_table("db.user_events", "s3://bucket/metadata/v1.metadata.json")
    
    # Initialize Engine coordinator
    engine = IcebergEngineCoordinator(catalog, s3, "db.user_events")
    
    # Verify Read Path (prunes non-matching ranges)
    # Search for partition=1 and ID >= 5 (Should read data_1.parquet)
    engine.execute_query(filter_partition=1, filter_key_min=5)
    
    # Verify Write Path Ingestion (Snapshot ID 2)
    # Appends new record Charlie in partition 1, keys 15-15
    engine.execute_write_append(1, [{"id": 15, "name": "Charlie"}], 15, 15)
    
    # Re-run Read Path (Should read both data_1 and data_2 parquet files)
    engine.execute_query(filter_partition=1, filter_key_min=5)
    
    # Simulate a concurrent collision retry loop!
    print("\n--- Simulating Concurrency Collision Loop ---")
    # We trigger two concurrent appends on the table.
    # The first append starts, then a second concurrent append commits, causing the first to rebase and retry.
    import threading
    
    def append_david():
        time.sleep(0.05) # Delay slightly
        engine.execute_write_append(2, [{"id": 20, "name": "David"}], 20, 20)
        
    def append_eve():
        engine.execute_write_append(2, [{"id": 30, "name": "Eve"}], 30, 30)

    t1 = threading.Thread(target=append_david)
    t2 = threading.Thread(target=append_eve)
    t1.start()
    t2.start()
    t1.join()
    t2.join()
    
    # Final Table Scan to verify all concurrent records survived
    engine.execute_query(filter_partition=2, filter_key_min=10)
